本章介绍Screeps Arena的三种输出方式及代码部署流程。

### Print
**基础输出，仅限tick≥1时生效。**
```python
from builtin import *


# ❌ 全局区域无效，不会显示
print("Hello, at global.")   # tick == 0


def init(k: GlobalKnowledge):
    print("Hello, at init.")   # tick == 1


def step(k: GlobalKnowledge):
    print("Hello, at step.")   # tick >= 1
```

### Jprint
<details>
<summary>
调试专用，显示JavaScript原生对象结构。
</summary>

与`print`对比：

| 数据类型 | print | jprint |
|----------|-------|--------|
| 简单类型(int/list) | `1 [1,2]` | `1 [1,2]` |
| 对象 | 格式化字符串 | 完整属性列表 |

**适用场景**：需查看对象全部细节时启用。

</details>

### Std 标准输出
<details>
<summary>
分级日志，标明调用者。
</summary>

**通用形参格式**：`(caller: str, msg: str)`
- `caller`：调用者名称，用于区分不同来源的日志
- `msg`：要输出的消息内容

```python
std.log("调用者", "普通日志")    # 灰色
std.info("调用者", "提示信息")   # 绿色
std.warn("调用者", "警告信息")   # 黄色
std.error("调用者", "致命错误")  # 红色并中断执行
```

</details>

### 编译与运行

**三步完成部署：**

1. **配置`build.py`**
   - 游戏内 → `External Editor` → 右键`main.mjs`复制路径
   - 粘贴到`build.py`的`config.target`（取消注释）
   - 可选：设置`config.language='cn'`启用部分汉化

2. **执行编译**
   - PyCharm中运行`build.py`
   - Python代码自动转译为JavaScript

3. **验证结果**
   - 点击`Play Game`
   - 成功 → 弹出`Pass`提示

**常见问题**：
- `[Compiler.compile Error]: 目标导出目录不存在`
  - 解决方法：确保`config.target`路径正确，且目录存在。
